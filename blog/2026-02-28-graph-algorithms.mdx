## 1. 入门阶段：图的表达与基础遍历

### 1.1 邻接表：Python 的优雅实现

对于大多数稀疏图（Sparse Graph），邻接表是性能与空间的最佳平衡。

```python
from collections import defaultdict

class Graph:
    def __init__(self):
        self.adj = defaultdict(list)

    def add_edge(self, u, v, weight=1):
        self.adj[u].append((v, weight))

# 这种结构 natural 地支持了 Python 的 for-in 循环
```

### 1.2 遍历逻辑的差异性

- **BFS (广度优先)**：利用队列分层扩展。它是寻找**无权图最短路径**的唯一最优解。
- **DFS (深度优先)**：利用递归或栈探寻边界。它是解决**拓扑排序**和**环检测**的核心。

## 2. 进阶阶段：路径优化与结构约束

### 2.1 Dijkstra 及其局限性

Dijkstra 虽强，但无法处理**负权边**。当面临金融风控中的“信用抵扣”或“回扣”场景（负值路径）时，我们需要 **Bellman-Ford** 或 **SPFA**。

### 2.2 拓扑排序 (Topological Sorting)

在处理流水线依赖（如 DAG 编译器）时，拓扑排序能给出执行的线性序列。

```python
def kahn_topo_sort(num_nodes, edges):
    in_degree = [0] * num_nodes
    adj = defaultdict(list)
    for u, v in edges:
        adj[u].append(v)
        in_degree[v] += 1

    queue = [i for i in range(num_nodes) if in_degree[i] == 0]
    result = []

    while queue:
        u = queue.pop(0)
        result.append(u)
        for v in adj[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.append(v)
    return result if len(result) == num_nodes else "Cycle Detected"
```

### 2.3 Bellman-Ford：处理负权边的利器

与 Dijkstra 不同，Bellman-Ford 可以处理包含负权边的图，并能检测**负权环**。这是非原生算法库中经常需要“手搓”实现的经典代码。

```python
def bellman_ford(vertices, edges, start):
    # edges: [(u, v, w), ...]
    dist = {v: float('inf') for v in vertices}
    dist[start] = 0

    # 进行 V-1 次松弛操作
    for _ in range(len(vertices) - 1):
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w

    # 第 V 次松弛用于检测负权环
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return "Graph contains negative weight cycle"

    return dist
```

### 2.4 Prim 算法：最小生成树 (MST)

在构建最低成本连接（如敷设网线、建立交通网络）时，Prim 算法能以最优成本覆盖所有节点。

```python
import heapq

def prim_mst(graph, start):
    # graph: {u: [(v, w), ...]}
    mst = []
    visited = {start}
    edges = [(w, start, v) for v, w in graph[start]]
    heapq.heapify(edges)

    while edges:
        w, u, v = heapq.heappop(edges)
        if v not in visited:
            visited.add(v)
            mst.append((u, v, w))
            for next_v, next_w in graph[v]:
                if next_v not in visited:
                    heapq.heappush(edges, (next_w, v, next_v))
    return mst
```

## 3. 高阶阶段：强连通与定量影响力

### 3.1 Tarjan 算法：寻找“小团体”

强连通分量 (SCC) 能识别图中那些能相互到达的闭环。在社交网络中，SCC 常被用于识别核心小团体。

### 3.2 PageRank：定量的价值衡量

PageRank 不仅仅是搜索引擎的专利。它在推荐系统中用于评估 User-Item 二部图中的节点权威性。

```python
# 简化的 PageRank 核心公式实现思路
def pagerank_step(nodes, pr_values, d=0.85):
    new_pr = {}
    N = len(nodes)
    for node in nodes:
        rank_sum = sum(pr_values[m] / len(out_neighbors[m])
                       for m in in_neighbors[node])
        new_pr[node] = (1 - d) / N + d * rank_sum
    return new_pr
```

## 4. 工业级深度解析：Neo4j Graph Data Science (GDS)

当你的数据量级跨越千万级节点时，“手搓”算法由于内存占用与并发管理的限制，往往难以支撑生产环境。此时，**Neo4j Graph Data Science (GDS)** 库成为了事实上的工业标准。它通过将图数据投影到内存中（Graph Projection），利用并行化引擎实现极致的计算性能。

基于官方文档的 19 个核心维度，我们将 GDS 算法库划分为以下五大实战矩阵：

### 4.1 中心性算法 (Centrality)：衡量节点权重与排名

此类算法旨在量化节点在网络中的“话语权”。

| 算法                  | 核心原理与公式                                                            | 典型应用与链接                                                                                                               |
| :-------------------- | :------------------------------------------------------------------------ | :--------------------------------------------------------------------------------------------------------------------------- |
| **PageRank**          | 基于链接传递权重：$PR(A) = \frac{1-d}{N} + d \sum \frac{PR(T_i)}{C(T_i)}$ | 网页排名、关键基础设施识别。 [View Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/page-rank/)             |
| **ArticleRank**       | PageRank 变体，通过降低低度数节点的初始贡献来缓解“蜘蛛陷阱”。             | 学术文献引用价值评估。 [View Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/article-rank/)                |
| **Degree Centrality** | 直接度量：计算节点的总入度/出度（权重图则算权重的总和）。                 | 社交网络人气指数、流量枢纽监测。 [View Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/degree-centrality/) |

### 4.2 社区发现与连通性 (Community & Connectivity)：挖掘拓扑结构

- **Leiden 算法**：Louvain 算法的现代改良版，通过最大化模块度（Modularity）发现层级社区。它修正了 Louvain 可能产生内部连通性差的社区的缺陷。 [Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/leiden/)
- **K-Core**：递归提取度数至少为 k 的子图节点。用于寻找网络中生存力最强、最核心的群体。 [Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/k-core/)
- **切断点 (Articulation Points)**：找出那些一旦移除就会导致图分裂成更多连通分量的“咽喉节点”。 [Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/articulation-points/)
- **WCC & SCC**：
  - **WCC (弱连通)**：忽略方向，识别“孤岛”数据。 [Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/wcc/)
  - **SCC (强连通)**：严格要求双向可达，识别紧密的互动闭环。 [Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/strongly-connected-components/)

### 4.3 相似性与搜索 (Similarity & Searching)：邻域发现与路经优化

| 算法                | 描述                                               | 应用场景与链接                                                                                                                                                                                                            |
| :------------------ | :------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **KNN (K-Nearest)** | 基于节点属性特征向量寻找最接近的 K 个邻居。        | 自动化推荐、特征补全。 [View Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/knn/)                                                                                                                      |
| **Node Similarity** | 基于 Jaccard 系数衡量节点邻居集合的交集程度。      | 好友推荐、内容相似性比对。 [Normal](https://neo4j.com/docs/graph-data-science/current/algorithms/node-similarity/) / [Filtered](https://neo4j.com/docs/graph-data-science/current/algorithms/filtered-node-similarity/)   |
| **A\* Search**      | 引入启发式函数 $f(n) = g(n) + h(n)$ 引导搜索方向。 | 指令集路径规划、游戏寻路。 [View Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/astar/)                                                                                                                |
| **Dijkstra [S-T]**  | 计算指定源节点到目标节点的精确最短路径。           | 网络路由、物流交付。 [View Doc](https://neo4j.com/docs/graph-data-science/current/algorithms/dijkstra-source-target/)                                                                                                     |
| **MST & Topo Sort** | 生成树优化与有向图拓扑序列生成。                   | 网络布线、编译器依赖分析。 [MST](https://neo4j.com/docs/graph-data-science/current/algorithms/minimum-weight-spanning-tree/) / [Topo](https://neo4j.com/docs/graph-data-science/current/algorithms/dag/topological-sort/) |

### 4.4 机器学习与图嵌入 (Node Embeddings)

图嵌入将高维拓扑结构映射为低维浮点向量，是图机器学习（GML）的基石。

1. **FastRP (Fast Random Projection)**：一种基于随机投影的可扩展算法。通过多次矩阵迭代更新，在极短时间内生成高质量的向量表示。 [Doc](https://neo4j.com/docs/graph-data-science/current/machine-learning/node-embeddings/fastrp/)
2. **Node2Vec**：结合了 BFS（提取局部结构）和 DFS（提取同质性）的游走模型。 [Doc](https://neo4j.com/docs/graph-data-science/current/machine-learning/node-embeddings/node2vec/)
3. **HashGNN**：利用消息传递（Message Passing）和局部敏感哈希（Min-Hashing）技术，捕捉复杂的节点特征。 [Doc](https://neo4j.com/docs/graph-data-science/current/machine-learning/node-embeddings/hashgnn/)

## 5. 架构选型与综合考量

1. **数据规模**：GB 级别选 **Neo4j**；TB/PB 级别首选 **Nebula Graph**。
2. **读写特性**：高频更新、低延迟查询选 **Memgraph**。
3. **语言偏好**：追求 SQL 感选 **Nebula**；追求类似 Python 的表现力选 **Neo4j (Cypher)**。

## 5. 总结与前瞻：图神经网络 (GNN)

图算法的演进并未停止。随着深度学习的发展，**图神经网络 (GNN)** 正在将算法从“手工设计规则”推向“表征学习”。

通过 **GraphSAGE** 或 **GAT**，我们现在可以让机器自动学习图中节点的特征向量（Node Embedding），从而在分类和预测任务中达到惊人的准确度。
